This glossary bridges three domains that intersect in Frontier Flow: **EVE Frontier** game mechanics, **Sui Move** blockchain concepts, and **Frontier Flow** application terms.

---

## EVE Frontier — Game Domain

| Term                                     | Definition                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Aggression**                           | An in-game event triggered when one rider attacks another. Represented by the `AggressionNode` in Frontier Flow, which emits `aggressor` and `victim` entity references.                                                                                                                                                                                                                                                                                                                                                                                             |
| **Tribe**                                | A player organization in EVE Frontier. Riders belong to tribes identified by numeric IDs. Used in friend-or-foe logic (e.g., `ListOfTribe` → `IsInList`).                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Priority List**                        | A `vector<ReturnTargetPriorityList>` of targets maintained by a Smart Turret. When target behaviour changes (entity enters proximity, starts attacking, or stops attacking), the on-chain `get_target_priority_list` function evaluates candidates and returns prioritized targets. The function receives a BCS-serialized `vector<TargetCandidate>` and returns a BCS-serialized `vector<ReturnTargetPriorityList>`. Represented by the `AddToQueueNode` in Frontier Flow.                                                                                          |
| **Priority Queue**                       | _(UI term)_ The Frontier Flow canvas uses "Priority Queue" as the user-facing label for the priority list concept, mapping to the `AddToQueueNode`. On-chain, the data structure is a `vector<ReturnTargetPriorityList>` priority list, not a traditional queue.                                                                                                                                                                                                                                                                                                     |
| **Proximity**                            | A detection event triggered when an entity enters the sensor range of a Smart Assembly. Represented by the `ProximityNode`.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Smart Assembly**                       | A programmable in-game structure (e.g., turrets, gates) whose behaviour is defined by Sui Move smart contracts. Frontier Flow generates the logic for these assemblies. Assemblies connect to a **NetworkNode** for energy, are anchored/unanchored by an admin, and can be brought online/offline by their owner via an **OwnerCap**. (Note: MVP focuses exclusively on **Turrets**).                                                                                                                                                                               |
| **Standing**                             | A numeric reputation value representing the relationship between two entities. Mapped to the `Value` socket type.                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Turret**                               | The primary type of Smart Assembly supported for the MVP (`world::turret::Turret`). A turret is anchored to a **NetworkNode**, brought online with energy reservation (via `online()`), and executes targeting via `get_target_priority_list`. Default rules: (1) exclude same tribe non-aggressors, (2) exclude STOPPED_ATTACK, (3) +10000 weight for STARTED_ATTACK, (4) +1000 weight for ENTERED (if different tribe or is aggressor). Owners can override targeting via an **extension contract** using the typed witness pattern (`authorize_extension<Auth>`). |
| **HP / Shield / Armor Ratio**            | Numeric `u64` values (0-100) on the `TargetCandidate` struct representing the remaining hit points (`hp_ratio`), shields (`shield_ratio`), or armor (`armor_ratio`) of a target. Used for priority ranking (e.g., targeting the weakest ship first).                                                                                                                                                                                                                                                                                                                 |
| **TargetCandidate**                      | The on-chain struct (`world::turret::TargetCandidate`) describing a potential target sent by the game to the turret. Fields: `item_id` (u64), `type_id` (u64), `group_id` (u64), `character_id` (u32), `character_tribe` (u32), `hp_ratio`, `shield_ratio`, `armor_ratio`, `is_aggressor` (bool), `priority_weight` (u64), `behaviour_change` (BehaviourChangeReason enum). Serialized via BCS as `vector<u8>` for transport to the turret's `get_target_priority_list` function.                                                                                    |
| **ReturnTargetPriorityList**             | The on-chain struct (`world::turret::ReturnTargetPriorityList`) returned by `get_target_priority_list`. Contains `target_item_id` (u64) and `priority_weight` (u64). The game shoots the target with the highest priority weight; if weights are equal, it shoots the first one in the list. Serialized via BCS as `vector<u8>` when returned from the extension.                                                                                                                                                                                                    |
| **BehaviourChangeReason**                | An enum (`world::turret::BehaviourChangeReason`) indicating why a target candidate was sent: UNSPECIFIED (0), ENTERED (1 - target entered proximity), STARTED_ATTACK (2 - target started attacking the base), STOPPED_ATTACK (3 - target stopped attacking). The game sends exactly one reason per candidate, prioritizing higher-priority reasons (e.g., STARTED_ATTACK over ENTERED when both apply).                                                                                                                                                              |
| **NetworkNode**                          | An on-chain energy provider (`world::network_node::NetworkNode`) to which Smart Assemblies are anchored. Turrets must be connected to an online NetworkNode to reserve energy and come online.                                                                                                                                                                                                                                                                                                                                                                       |
| **OwnerCap**                             | An owned capability object (`world::access::OwnerCap<T>`) granting the holder permission to operate on their assembly (e.g., bring a turret online/offline, authorise extensions). Transferred to the owner's character on anchor.                                                                                                                                                                                                                                                                                                                                   |
| **OnlineReceipt**                        | A hot-potato proof struct (`world::turret::OnlineReceipt`) returned by `verify_online()`. Proves the turret was online at the point of invocation and must be consumed within the same transaction.                                                                                                                                                                                                                                                                                                                                                                  |
| **Extension (Typed Witness)**            | A builder-defined Move module that overrides default turret targeting logic. Registered via `authorize_extension<Auth>()` using a `drop`-capable witness struct. When an extension is configured, the world contract's default `get_target_priority_list` aborts with `EExtensionConfigured` (error code 7), and the game server resolves the package ID from the configured type name and calls the extension's `get_target_priority_list` function directly.                                                                                                       |
| **BCS (Binary Canonical Serialization)** | Sui's standard serialization format used to encode/decode `TargetCandidate` and priority lists as `vector<u8>` for cross-module transport. The turret contract provides `unpack_candidate_list()` / `unpack_priority_list()` to deserialize `vector<TargetCandidate>` and `unpack_return_priority_list()` for deserialization. Extension functions accept and return BCS bytes for game server compatibility.                                                                                                                                                        |

## Sui Move — Blockchain Domain

| Term               | Definition                                                                                                                                                                                                                                                                                             |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Entry Function** | A `public entry fun` in Sui Move that can be called directly by a transaction. Frontier Flow generates entry functions for event triggers (e.g., `in_proximity`, `on_aggression`). Note: the turret extension pattern uses `public fun` (not `entry`) because the game server composes calls via PTBs. |
| **Gas**            | The computational cost of executing operations on the Sui blockchain. Measured in MIST (1 SUI = 10⁹ MIST). The AST optimiser in Frontier Flow aims to minimise gas costs.                                                                                                                              |
| **Module**         | A Sui Move compilation unit containing struct definitions, functions, and imports. Each Frontier Flow graph generates one module (e.g., `builder_extensions::turret_logic`).                                                                                                                           |
| **Object**         | A Sui-native concept where data is stored as objects with unique IDs. Relevant for `UpgradeCap` and deployed package references.                                                                                                                                                                       |
| **Package**        | A published collection of Sui Move modules. When Frontier Flow deploys, it publishes a package to the Sui network.                                                                                                                                                                                     |
| **Sui**            | A Layer 1 blockchain with a Move-based smart contract language. Frontier Flow targets the Sui network for deploying generated contracts.                                                                                                                                                               |
| **TxContext**      | A special Sui Move parameter (`&mut TxContext`) passed to entry functions, providing transaction metadata (sender, epoch, etc.).                                                                                                                                                                       |
| **UpgradeCap**     | A Sui object that grants the holder permission to upgrade a published package. Created on first deploy, stored in IndexedDB and optionally GitHub. Loss of the `UpgradeCap` makes the package permanently non-upgradable ([R-08](./RISK-REGISTER.md)).                                                 |
| **UpgradePolicy**  | Determines what changes are allowed during a package upgrade. `Compatible` (default) requires struct layouts and public function signatures to remain stable.                                                                                                                                          |

## Frontier Flow — Application Domain

| Term                                 | Definition                                                                                                                                                                                 |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **AST Pruning**                      | An optimisation pass (Phase 3.5) that eliminates dead branches, folds redundant vectors, and propagates constants in the intermediate representation before code emission.                 |
| **Canvas**                           | The main interactive area powered by React Flow where users drag, drop, and connect nodes.                                                                                                 |
| **Code Generator**                   | The planned `codeGenerator.ts` module that will transform a visual node graph into Sui Move source code through a multi-phase pipeline.                                                    |
| **Compilation Pipeline**             | The 5-phase transformation: Graph → IR → Constraint Validation → Sanitisation → AST Optimisation → Move Emission.                                                                          |
| **Constraint Engine**                | Phase 2 of the pipeline. Validates graph completeness (no disconnected subgraphs, all required inputs connected) before allowing code generation.                                          |
| **DnDFlow**                          | The internal component in `App.tsx` that wraps React Flow with drag-and-drop handlers for adding nodes from the sidebar.                                                                   |
| **Edge**                             | A connection between two node sockets on the canvas. Edges are colour-coded by socket type and animated.                                                                                   |
| **Emitter**                          | Phase 4 of the pipeline. Converts the validated IR into Move source code text, annotating each line with `@ff-node:` comments for error traceability.                                      |
| **Handle**                           | A React Flow `<Handle>` component representing a socket's visual connection point on a node.                                                                                               |
| **IR (Intermediate Representation)** | A structured, normalized representation of the node graph used internally by the code generation pipeline. See [API-CONTRACTS.md §2](./API-CONTRACTS.md#2-intermediate-representation-ir). |
| **Node**                             | A visual block on the canvas representing a game logic element (event trigger, data lookup, logic gate, or action).                                                                        |
| **Node Type Registry**               | The `src/nodes/index.ts` mapping from type strings (e.g., `"proximity"`) to React components.                                                                                              |
| **Sidebar**                          | The right-side panel containing draggable node definitions that users can drop onto the canvas.                                                                                            |
| **Socket**                           | A typed input or output port on a node. Sockets enforce connection rules based on their data types (Blender-style paradigm).                                                               |
| **Source Map**                       | A mapping from emitted Move source line numbers to originating React Flow node IDs, used for compiler error traceability.                                                                  |
| **Toast**                            | A notification popup (success, error, info) displayed to the user for compilation progress, deployment results, and error messages.                                                        |

## Socket Type Mapping

Quick reference mapping the three domain layers:

**On-Chain Mapping:** In the `world::turret` contract, socket types map to `TargetCandidate` fields and accessor functions. The `rider` socket maps to `character_id` (u32), `tribe` maps to `character_tribe` (u32), and `number`/`value` sockets map to ratio fields (`hp_ratio`, `shield_ratio`, `armor_ratio`) and `priority_weight` (u64). The `target` socket maps to `item_id` and `type_id`.
