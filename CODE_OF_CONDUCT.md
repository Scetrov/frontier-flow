# Code of Conduct

## Our Commitment

Frontier Flow is maintained as a respectful, constructive, and technically serious open-source project. We want contributors, users, and reviewers to feel welcome participating in discussions, submitting issues, and proposing improvements.

## Expected Behavior

Participants in this project are expected to:

- communicate respectfully and assume good intent
- keep feedback specific, technical, and actionable
- welcome different experience levels and perspectives
- focus on improving the project rather than winning arguments
- respect maintainer time, review bandwidth, and project boundaries

## Unacceptable Behavior

The following behavior is not acceptable in project spaces, including issues, pull requests, reviews, and discussions:

- harassment, insults, or personal attacks
- discriminatory, hateful, or demeaning language
- deliberate disruption, trolling, or bad-faith escalation
- publishing private or sensitive information without permission
- repeated disregard for project rules, review feedback, or moderation decisions

## Scope

This code of conduct applies across all project-managed spaces, including:

- GitHub issues
- pull requests
- code reviews
- community discussions related to Frontier Flow

## Reporting

If you experience or witness behavior that violates this code of conduct, contact the maintainer through [scetrov.live](https://scetrov.live) and include as much relevant context as possible.

## Enforcement

The maintainer may remove content, lock discussions, reject contributions, or restrict participation when behavior harms the project or its community.

Enforcement decisions will be made with the goal of protecting contributors, preserving a productive working environment, and keeping technical collaboration possible.

## Project Standards

Technical disagreement is expected. Disrespect is not.

We encourage rigorous debate, strong opinions, and high engineering standards, provided they are expressed professionally.
