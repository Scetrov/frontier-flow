# 1. Data Model

## 1.1 BundledDependencySnapshot

| Field | Type | Notes |
| --- | --- | --- |
| `sourceVersionTag` | `string` | Matches the world-contract version in `packageReferences.ts`. |
| `resolvedAt` | `number` | Epoch milliseconds when the preload script produced the snapshot. |
| `resolvedDependencies.files` | `string` | Serialized extra root file payloads; currently not required for this feature. |
| `resolvedDependencies.dependencies` | `string` | Serialized array of package snapshots used to rebuild the local dependency tree. |
| `resolvedDependencies.lockfileDependencies` | `string` | Serialized lockfile-resolution metadata returned by the builder. |

**Validation Rules**:

- `sourceVersionTag` MUST equal the selected target's `sourceVersionTag` before runtime reuse.
- `resolvedDependencies.dependencies` MUST parse into an array.
- The parsed package set MUST contain `MoveStdlib`, `Sui`, and `World` with non-empty file maps for supported remote targets.

## 1.2 ResolvedDependencyPackageSnapshot

| Field | Type | Notes |
| --- | --- | --- |
| `name` | `string` | Package name from the builder output; matching should be case-insensitive. |
| `files` | `Record<string, string>` | Source file payloads keyed by dependency-relative path. |

**Relationships**:

- Many `ResolvedDependencyPackageSnapshot` records belong to one `BundledDependencySnapshot`.
- The package snapshots are the source material for `MaterializedDependencyTree`.

## 1.3 MaterializedDependencyTree

| Field | Type | Notes |
| --- | --- | --- |
| `files` | `Record<string, string>` | Final file map passed to `buildMovePackage()`. |
| `packageMap` | `Record<string, string>` | Deterministic mapping from package identity to local dependency path such as `deps/world` or `deps/sui`. |
| `rewritesApplied` | `readonly string[]` | Describes manifest rewrites and sanitization applied while materializing packages. |

**Validation Rules**:

- The root extension package MUST still point to the live `world` package ID semantics already enforced by deploy-grade compilation.
- The materialized tree MUST be complete enough that `buildMovePackage()` does not need live GitHub package fetches on a snapshot hit.

## 1.4 DeployCacheManifestEntry

| Field | Type | Notes |
| --- | --- | --- |
| `sourceVersionTag` | `string` | Version tag to resolve. |
| `repositoryUrl` | `string` | Currently `https://github.com/evefrontier/world-contracts.git`. |
| `subdirectory` | `string` | Root package path, currently `contracts/world`. |
| `outputPath` | `string` | Checked-in snapshot destination under `public/`. |
| `targets` | `readonly string[]` | Supported targets that share the version tag. |

**Relationships**:

- One `DeployCacheManifestEntry` produces one `BundledDependencySnapshot`.
- Multiple targets may share the same entry when they use the same `sourceVersionTag`.

## 1.5 SnapshotValidationResult

| Field | Type | Notes |
| --- | --- | --- |
| `isValid` | `boolean` | Whether the snapshot is safe to use for cache-first deploy-grade compilation. |
| `missingPackages` | `readonly string[]` | Required packages not present in the snapshot. |
| `emptyPackageNames` | `readonly string[]` | Packages present without usable file payloads. |
| `fallbackAllowed` | `boolean` | Whether runtime may fall back to the network-backed path. |
| `message` | `string` | Human-readable validation summary for preload or runtime diagnostics. |

## 1.6 State Transitions

1. `DeployCacheManifestEntry` is configured.
2. `BundledDependencySnapshot` is generated by the preload script.
3. `SnapshotValidationResult` marks the snapshot valid or invalid.
4. Runtime loader selects a valid snapshot for the requested target and tag.
5. `MaterializedDependencyTree` is built from parsed package snapshots.
6. Deploy-grade compilation either succeeds from the materialized tree or falls back to the network-backed path if validation rejects the snapshot.
